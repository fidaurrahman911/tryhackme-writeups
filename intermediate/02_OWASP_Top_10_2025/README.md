# OWASP Top 10 2025

**Difficulty:** Intermediate

**Platform:** TryHackMe

---

## Objective

The objective of this room was to understand the most common web application security vulnerabilities based on the OWASP Top 10 framework. The room introduced modern attack categories, explained how these vulnerabilities occur, and demonstrated their impact on web application security.

---

## Topics Covered

- Identification and Authentication Failures (IAAA)
- Insecure Data Handling
- Broken Access Control
- Cryptographic Failures
- Injection Attacks
- Security Misconfiguration
- Vulnerable Components
- Software and Data Integrity Failures
- Security Logging and Monitoring Failures
- Server-Side Request Forgery (SSRF)

---

## What I Learned

During this room I learned that web application security depends on much more than writing functional code. Developers must also implement authentication correctly, protect sensitive information, validate user input, monitor system activities, and continuously update software components.

I also learned that many successful cyber attacks exploit simple security mistakes rather than advanced hacking techniques. Understanding these vulnerabilities helps security professionals identify weaknesses before attackers can exploit them.
