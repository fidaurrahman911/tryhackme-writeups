**Difficulty:** Beginner

**Platform:** TryHackMe

# Objective

The objective of this room was to understand how the Hypertext Transfer Protocol (HTTP) and HTTPS work, how web browsers communicate with servers, and why secure communication is essential for modern web applications.

---

# What I Learned

During this room I learned how to:

- Understand the HTTP request and response lifecycle.
- Differentiate between HTTP and HTTPS.
- Learn how SSL/TLS certificates secure web communications.
- Recognize common HTTP methods and status codes.
- Understand how browsers establish secure connections with web servers.

---

# Practical Skills

## HTTP Communication

Learned how a client sends requests to a web server and receives responses.

Covered concepts such as:

- Request headers
- Response headers
- HTTP methods (GET, POST, PUT, DELETE)
- Status codes
- URLs and resources

---

## HTTPS & Certificates

Explored how HTTPS encrypts communication between clients and servers using SSL/TLS.

Learned how browsers verify certificates and display warnings when certificates are invalid or untrusted.

- HTTP Fundamentals

# Key Takeaways

Understanding HTTP and HTTPS is fundamental for every cybersecurity professional. Secure communication protects users from eavesdropping, data tampering, and impersonation attacks. Knowledge of HTTP requests, responses, certificates, and secure connections provides a strong foundation for web application security and penetration testing.
